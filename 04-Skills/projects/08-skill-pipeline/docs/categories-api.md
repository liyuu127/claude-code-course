The Categories API provides endpoints for managing product categories. Categories can be listed and viewed by anyone, but creation, updating, and deletion require authentication and admin privileges.

### GET `/:id`

**Description**: Get a specific category with its associated products (public access)

**Authentication**: Not required

**Parameters**:

| Name | In | Type | Required | Description |
|------|----|------|----------|-------------|
| id | path | string | Yes | The category identifier |

**Responses**:

| Status | Description |
|--------|-------------|
| 200 | Successful response with category and products |
| 404 | Category not found |

**Response Example**:
```json
{
  "data": {
    "id": "cat_1",
    "name": "Electronics",
    "description": "Electronic devices and accessories",
    "products": [
      {
        "id": "prod_1",
        "name": "Smartphone",
        "price": 599.99
      }
    ]
  }
}
```

**Error Response Example**:
```json
{
  "error": "Category not found"
}
```

| Name | In | Type | Required | Description |
|------|----|------|----------|-------------|
| name | body | string | Yes | The category name |
| description | body | string | No | The category description |

```json
}
```

### DELETE `/:id`

**Description**: Delete a category (admin only)

**Authentication**: Required - User must be authenticated and have admin role

**Middleware**: `requireAuth`, `isAdmin`

**Parameters**:

| Name | In | Type | Required | Description |
|------|----|------|----------|-------------|
| id | path | string | Yes | The category identifier |

**Responses**:

| Status | Description |
|--------|-------------|
| 204 | Category deleted successfully |
| 401 | Authentication required |
| 403 | Admin access required |
| 404 | Category not found |
| 409 | Cannot delete - category has existing products |

**Error Response Example** (Conflict):
```json
{
  "error": "Cannot delete category with existing products"
}
```
